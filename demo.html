老师，接下来我将进行演示效果展示。为了强化呈现效果，以下数据为模拟生成，旨在展示系统在实际拥有数据时的洞察能力。真实场景中，系统严格基于实际行为数据进行分析，不会呈现未获取的数据维度。

Roadshow Script

现场演示按“权限可信 → 组织盘点 → 个人诊断 → 异常治理 → 扩展组队”推进

1团队人才盘点

业务管理者看智能教学产品部晋升准备度，快速展示准备度分布、关键人才和风险预警。

2组织部盘点

战略层 / HR 看组织健康度、人才密度、九宫格、异常案例和行动建议。

3个人晋升诊断

个人视角只看本人准备度、能力雷达、成长建议，不展示团队 360。

4异常人才识别

HR 专家看脱敏异常案例、归因分析和干预建议。

5权限对比

同一个“看团队准备度”的问题，员工被拒绝，管理者通过。

这条动线要让评委看到：Talent-Pilot 不是把一堆数据摆出来，而是把不同角色的决策任务组织成可验证、可追溯、可复核的判断过程。

Script Scene 01

业务管理者：智能教学产品部晋升准备度

这是脚本里的第一段实机演示，适合用来快速建立“身份验证通过后，系统能生成团队人才盘点报告”的产品信任。

【身份验证】
工号：289545
部门：组织发展
职位：组织发展部部长

【请求】
帮我看看智能教学产品部的晋升准备度

讲解重点

  • 先展示身份验证通过与“可查看团队数据”的权限范围。
  • 输出智能教学产品部准备度分布、Top 高潜、评价偏差与状态异常。
  • Aha：团队盘点结果一键生成，管理者从找数据转向看风险与行动建议。
脚本输出模块 团队概况 / 准备度分布 / 关键人才 / 风险预警 / 行动建议

Scene 01

组织部部长：人才盘点与异常治理

证明 Talent-Pilot 能从组织负责人视角生成团队健康度、人才密度、九宫格、异常案例和行动建议。

【身份验证】
工号：100001
部门：组织部
职位：组织部部长

【请求】
请为我展示组织部的人才盘点情况，我需要了解：
1. 团队整体健康度和人才密度
2. 九宫格人才落位分布
3. 有哪些需要关注的异常人才案例
4. 针对盘点结果的行动建议

讲解重点

  • 先验证身份和权限，组织负责人可看团队聚合盘点。
  • 重点展示低调高贡献、高曝光证据不足、评价分歧三类异常。
  • Aha：人才盘点不是平均扫一遍，而是把管理注意力集中到争议地带。
输出模块 健康度 / 人才密度 / 九宫格 / 异常案例 / 行动建议
进入实机
Scene 02

HRBP：资深 HRBP 个人晋升诊断

证明同一套数据底座面对个人角色时，会生成成长视角的晋升准备度，同时保护 360 与他人信息。

【身份验证】
工号：180001
部门：组织部
职位：HRBP

【请求】
请为我生成个人晋升诊断报告，目标岗位是资深HRBP。我想了解：
1. 我当前的晋升准备度评分
2. 在组织中的百分位定位
3. 能力雷达图与目标岗位的差距
4. 晋升概率预测
5. 需要重点提升的能力维度

讲解重点

  • 目标岗位不是外部职级，用内部岗位能力要求和 1.1-7.3 职级口径解释。
  • 展示个人准备度、百分位、能力差距和补证据建议。
  • Aha：员工看到的不是裁判结果，而是离目标岗位还差哪些可补充证据。
输出模块 准备度 / 百分位 / 雷达图 / 晋升概率 / 能力提升建议
进入实机
Script Scene 03

一线员工：个人晋升准备度诊断

这个脚本场景用于强调个人权限边界：员工只看本人数据、直线上级反馈与成长建议，不看团队 360 或他人明细。

【身份验证】
工号：200001
部门：智能教学产品部
职位：算法工程师

【请求】
我想看看自己晋升2.3的准备度

讲解重点

  • 输出当前 2.2 → 目标 2.3 的档内晋升准备度。
  • 展示能力雷达、团队百分位、自评 vs 上级反馈差异。
  • Aha：个人端不是“判你行不行”，而是把目标差距翻译成可行动成长路径。
脚本输出模块 综合评估 / 能力雷达 / 上级反馈 / 自评对比 / 成长建议
进入实机
Script Scene 04

HR 专家：智能教学产品部异常人才识别

这个脚本场景用于展示 HR 专家如何基于脱敏数据识别自评过高、他评分歧和活跃度下降等异常。

【身份验证】
工号：300001
部门：人力资源部
职位：HRBP

【请求】
帮我识别智能教学产品部中的异常人才

讲解重点

  • 强调仅 HR 专家 / 战略层可查看脱敏异常案例。
  • 展示自评过高、他评分歧、流失风险三类异常归因。
  • Aha：HRBP 不是被数据淹没，而是直接看到要优先介入的案例。
脚本输出模块 异常案例 / 偏差分析 / 可能原因 / 风险等级 / 干预建议

Scene 03

HRBP 对接人：算法工程部异常分析

证明 Talent-Pilot 可以帮助 HRBP 从半年度盘点中识别异常类型、影响范围、可能原因和治理动作。

【身份验证】
工号：380001
部门：人力资源部
职位：HRBP（算法工程部对接人）

【请求】
请帮我分析一下AI技术中台算法工程部半年度人才盘点中的异常情况。

讲解重点

  • HRBP 有对接组织的脱敏统计权限，但仍不能查看无授权原文。
  • 重点看自评高于他评、低可见贡献、主管推荐与证据链不一致。
  • Aha：HRBP 更快知道哪里值得介入、为什么介入、怎么介入。
输出模块 异常总览 / 脱敏案例 / 组织洞察 / 治理建议
进入实机
Scene 04

高级项目经理：微前端架构升级智能组队

证明 Talent-Pilot 不只是 HR 场景，也能服务业务项目资源调配，从“找熟人”走向“找对人”。

【身份验证】
工号：400001
部门：项目管理中心
职位：高级项目经理

【项目需求】
项目类型：微前端架构升级项目
项目复杂度：高
所需角色：前端技术专家 × 2、后端接口工程师 × 1、产品经理 × 1
项目周期：3个月

【请求】
请基于以上需求为我推荐最优团队配置，并说明：
1. 推荐人员及匹配度评分
2. 团队协作风格互补性分析
3. 历史项目协作关联度
4. 潜在风险预警

讲解重点

  • 系统先解析项目需求，再匹配能力标签、项目经历、协作网络和可用性。
  • 重点不是谁最强，而是这几个人放在一起能否形成互补团队。
  • Aha：项目组队从熟人网络，升级为基于组织能力证据与协作关系的配置建议。
输出模块 推荐人员 / 互补性 / 历史协作 / 风险预警
进入实机
Permission Appendix

同一问题，不同角色，系统必须给出不同回答

这是答辩时最值得主动展示的可信边界：权限拒绝不是失败，而是 Talent-Pilot 可以进入组织真实流程的前提。

一线员工

“我想看看我们团队的晋升准备度”

系统拒绝：您只能查看自己的晋升准备度数据。
团队管理者

“帮我看看智能教学产品部的晋升准备度”

系统通过：展示完整团队人才盘点报告。
